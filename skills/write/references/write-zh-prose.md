# 中文短规则

默认中文 prose 先读这个。长文或复杂 AI 味再读 `write-zh.md`。

## 核心原则

- 删掉空泛开场，直接进入信息。不要写“X 的核心想法很简单”“说到底，X 就是……”“一个常见的误解是……”。
- 保留作者自己的词和节奏。
- 少用总结腔：总的来说、值得注意的是、可以说、在某种程度上。
- 少用排比式空话：不仅...而且...、既...又...、从...到...全方位。
- 少用假深度句式：不在...而在...、X 不仅仅是 Y、X 的用途早已超出 Y。两边的主张如果成立，直接各说各的。
- 把抽象判断落成具体事实。
- 少用破折号和括号，用逗号、句号或调整句子结构代替。
- 中文正文用全角引号 “ ” 和 ‘ ’，不用半角 " " 和 ' 。
- 不加粗强调，不把所有并列短语拆成列表。需要交叉对比的数据才用表格，能用句子讲清的就用句子。

## 常见改法

- “提供了一个完整的解决方案” -> “解决了...”
- “显著提升用户体验” -> 说清提升了什么体验。
- “让整个流程更加顺畅” -> 说清少了哪一步、快在哪里。
- “在复杂场景下依然表现稳定” -> 给场景或删掉。

## 保留

- 作者有意的口语。
- 必要的技术术语。
- 明确的情绪和立场。
- 已经自然的短句。
