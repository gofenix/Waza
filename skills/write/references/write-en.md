# English Writing Reference

英文编辑的目标不是“更高级”，而是更自然、更准确、更像真实作者。

## Common AI Smells

- Over-signposting: "In today's fast-paced world", "It is important to note".
- Fake balance: "not only... but also" when the sentence only needs one claim.
- Abstract nouns replacing actions: "implementation of improvements" instead of "improve".
- Inflated adjectives: robust, seamless, innovative, powerful, comprehensive.
- Repeated conclusions at the end of every section.
- Em-dash and parenthesis overuse. Use sentence rhythm, commas, and periods instead.
- Over-formatting: bold emphasis scattered through body text, bullet-pointing ordinary narrative, subheading every paragraph, and tabulating information that a sentence could carry.

## Editing Moves

- Cut the throat-clearing first sentence.
- Prefer verbs over noun stacks.
- Use the author's concrete nouns, not generic labels.
- Keep short sentences short.
- Remove recap sentences that add no new information.
- Keep technical terms exact.

## Before / After

Weak:

```text
This comprehensive solution enables users to seamlessly manage their workflow.
```

Better:

```text
Users can manage the workflow in one place.
```

Weak:

```text
It is important to note that this feature is not only powerful but also easy to use.
```

Better:

```text
This feature is useful because it removes one setup step.
```

## Keep

- Deliberate technical precision.
- Author voice, including blunt or casual phrasing.
- Product names, commands, paths, code identifiers.
- Necessary hedging when evidence is uncertain.

## Output

Return the edited text only unless the user asks for notes.
