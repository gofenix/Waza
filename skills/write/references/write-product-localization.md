# Product Localization

用于产品 UI、网站、应用商店、设置项、按钮、空状态、错误提示和多语言文案。

## Review Axes

- **Truth**：文案承诺是否和真实产品一致。
- **Fit**：按钮、tab、toast、menu 是否放得下。
- **Action**：错误和空状态是否告诉用户下一步。
- **Tone**：语气是否符合产品类型，工具型产品不要营销腔。
- **Consistency**：同一概念在不同语言和页面里是否同名。
- **Risk**：是否夸大安全、性能、隐私、自动化能力。

## Chinese UI Copy

- 按钮用动词短语：保存、导出、重试、继续。
- 状态文案少解释，多告诉用户现在发生什么。
- 错误提示写“为什么失败 + 下一步”，不要只写“操作失败”。
- 中文按钮过长时优先改短 copy，不要只缩字号。

## English UI Copy

- Prefer plain verbs: Save, Export, Retry, Continue.
- Avoid "seamless", "effortless", "powerful" unless evidence matters.
- Error copy should name the failed action and recovery path.
- Settings labels should be scannable, not sentence-long.

## Bilingual Consistency

- 数字、版本、日期、限制、价格、能力范围必须一致。
- 中英文不必逐字对应，但事实和承诺要对应。
- Release items 一一对应时，编号和意义要保持一致。
