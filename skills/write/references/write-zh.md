# 中文去 AI 味模式库

这是完整中文参考。只在短规则不够时读取，不要把它当逐条 checklist。

## 1. 空泛开场

删掉不能提供新信息的开场：

- “在当今快速发展的时代”
- “随着技术的不断进步”
- “这是一个非常重要的话题”
- “X 的核心想法很简单”
- “说到底，X 就是……”
- “一个常见的误解是……”

直接进入主题。

## 2. 总结腔

常见气味：

- “总的来说”
- “值得注意的是”
- “可以看出”
- “这不仅...也...”
- “从某种意义上说”
- “不在...而在...”
- “X 不仅仅是 Y”“X 的用途早已超出 Y”

如果句子只是重复上一段，删除；如果有新信息，改成具体判断。

## 3. 抽象名词堆叠

把“能力建设、体验优化、效率提升、流程闭环”拆成动作和结果。

差：

```text
该方案实现了对用户体验的全面优化。
```

好：

```text
用户少填两项信息，也不用在两个页面之间切换。
```

## 4. 没有证据的形容词

“显著、极大、全面、稳定、高效、丝滑、强大”如果没有证据，删掉或换成事实。

## 5. 复读表格

表格已经列出事实时，正文不要逐行复述。正文应该解释趋势、例外、判断或下一步。

## 6. 长文结构重复

长文里同一套判断反复出现时，合并到一个章节。不要每节都用“背景 -> 问题 -> 方案 -> 总结”的模板。

## 7. 中英混排

中文和英文术语之间留空格，但代码、路径、命令用反引号。不要把清楚的英文术语翻成生造中文。中文正文用全角引号 “ ” 和 ‘ ’，不用半角 " " 和 ' 。

## 8. 保留作者声音

不要把有个性的句子磨平。作者明确、尖锐、口语化时，只修不自然和不清楚的地方。

## 9. 删段前确认信息量

整段看起来重复时，先确认里面是否藏着唯一事实、数字、例子或限制。没有唯一信息再删。

## 10. 输出纪律

默认只输出改好的文本。用户要解释时，才给修改说明。不要在改稿后再加“希望对你有帮助”。

## 11. 过度格式强调

AI 写作倾向频繁使用 **加粗**、列表和多级标题来制造视觉层级，普通叙述被拆成说明书。

- 正文中用句子自身节奏传递重点，不要用加粗强调。
- 列表只用于必须在视觉上区分的并列信息，不用来装饰普通叙述。
- 连续段落不需要每段一个小标题。
- 能用一两句话说清的信息不做成表格。表格适合交叉对比的数据，不适合装饰普通叙述。
- 少用破折号和括号。用逗号、句号或调整句子结构来组织信息，不要用括号插入补充说明。
