# 中文 Release Notes

Release notes 写给工程师和用户，不写广告。

## 结构

- 每条一句话。
- 先写模块或技能名，再写变化和影响。
- 英文和中文条目要一一对应。
- 5 到 8 条足够，除非 release 很小或很大。

## 好条目

```text
1. **Check**: 发布前检查现在会验证 Codex plugin mirror，避免 release asset 漏带技能文件。
```

## 坏条目

```text
1. 全面优化体验，带来更稳定高效的使用感受。
```

问题：没有对象、没有变化、没有影响。

## 风格

- 工程师可读，短，具体。
- 不夸大，不写“重磅”“全新升级”“极致体验”。
- 不把内部重构写成用户价值，除非用户能感知。
- 不使用 em-dash。
