# 中文与双语审阅

用于中英混排、翻译审校、双语 release notes、产品本地化。

## 中文排版

- 中文和英文、数字之间通常留空格：`使用 Codex 处理 3 个任务`。
- 命令、路径、代码标识符用反引号，不强行加空格。
- 中文标点用于中文句子，英文标点用于纯英文片段。
- 中文正文用全角引号 “ ” 和 ‘ ’，不用半角 " " 和 ' 。纯英文片段内部的引号仍用英文标点。
- 品牌名、模型名、API 名保持原文。

## 翻译原则

- 先保事实，再保语气，最后才保句式。
- 不逐字硬译。英文名词堆叠要拆成中文动作或关系。
- 产品文案要像目标语言原生写作，不像翻译稿。
- 技术词首次出现可以写“中文解释（English term）”。

## Release Notes

- 中英文条目编号一一对应。
- 每条一句话，说清“改了什么 + 用户影响”。
- 英文不要夸张，中文不要口号化。
- 不把内部实现细节写成用户价值，除非它影响用户。
