# Logging Techniques

日志是为了证明假设，不是为了制造噪声。

好日志包含：

- 输入摘要，不泄露 secret。
- 唯一请求 ID、任务 ID 或样本 ID。
- 状态变化前后值。
- 分支选择原因。
- 耗时和重试次数。
- 错误类型和上游响应摘要。

调试步骤：

1. 先写出要证明的假设。
2. 只在关键边界打点：输入、解析、状态更新、外部调用、输出。
3. 保留时间戳和 ID，方便串联。
4. 验证后删除临时日志，或把有价值的日志降级成正式观测点。

不要记录：

- token、cookie、authorization header。
- 完整用户数据或内部 URL。
- 超大 payload。
- 会改变时序的重型日志。
